Conclusion

This is impressive. With a couple of queries, ChatGPT was able to analyze a document and provide output that can be sent to another process, such as an alert or audit log. This scenario demonstrates an initial step in exploring how AI can be used to identify sensitive data that may be stored in unexpected locations. It highlights AI's potential in improving usability for end users and its ability to detect data patterns indicating sensitive information.

The process was intuitive, allowing queries in sentence form that produced actionable results without requiring knowledge of a programming language. It was quick to generate the query and obtain feedback. The flexibility of adjusting queries based on the results also made evolving the use case seamless. The deliverable produced was actionable and met the required needs swiftly. AI is proving to be a powerful tool and resource that can help mitigate the risk of data loss and misuse within an organization, and this is only scratching the surface of its capabilities.


Takeaways

  • Create a file with text and add sensitive data that is broken up to see if ChatGPT can be used to identify data that is being hidden.
  • Try creating other types of sensitive data to see if it can pick up on it as well. (Please note that you should not use real sensitive data).
  • Try different output formats to see what ChatGPT can create.
