4.2.2 Types of Cryptology

Cryptology encompasses two primary disciplines: cryptography and cryptanalysis. Within these disciplines, there are various types and methods used to secure communications and break codes. Cryptography is the practice of designing algorithms and systems for secure communication in the presence of third parties with potentially malicious intent. It involves creating methods to encrypt and decrypt information, ensuring that data can be securely transmitted, stored, and accessed only by intended parties.

Symmetric-Key Cryptography

    • Definition:

      In symmetric-key cryptography, the same key is used for both encryption and decryption. This key must be shared between the communicating parties in a secure manner.

    • Use Case:

      Symmetric-key cryptography is fast and efficient, making it suitable for encrypting large volumes of data or for use in systems where processing power is limited.

    • Examples:

      AES (Advanced Encryption Standard) and DES (Data Encryption Standard).

Asymmetric-Key Cryptography (Public Key Cryptography)

    • Definition:

      Asymmetric-key cryptography uses a pair of keys: a public key, which can be shared openly, for encryption, and a private key, kept secret by the owner, for decryption. This solves the problem of key distribution that plagues symmetric-key cryptography.

    • Use Case:

      It's widely used for secure key exchange, digital signatures, and encrypting small amounts of data.

    • Examples:

      RSA (Rivest-Shamir-Adleman), ECC (Elliptic Curve Cryptography).

Hash Functions

    • Definition:

      Hash functions take an input (or 'message') and return a fixed-size string of bytes, typically a digest that is unique to each unique input. They are designed to be one-way functions, making it infeasible to invert or find two different inputs that produce the same output.

    • Use Case:

      Hash functions are used for integrity verification, password storage, and digital signatures.

    • Examples:

      SHA-256 (Secure Hash Algorithm 256-bit), MD5 (though considered insecure for many purposes now).
